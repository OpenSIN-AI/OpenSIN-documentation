# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of OpenSIN seriously. If you believe you have found a security vulnerability, please report it to us as described below.

### Security Response Process

1. **Report**: Send an email to [security@opensin.ai](mailto:security@opensin.ai) with details about the vulnerability
2. **Acknowledge**: We will acknowledge receipt of your report within 48 hours
3. **Investigate**: Our security team will investigate and validate the report
4. **Remediate**: We will work on a fix and coordinate with you on disclosure
5. **Disclose**: We will publicly disclose the issue after a fix is available

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

### Bug Bounty

We reward responsible disclosure of critical security vulnerabilities.

### Security Best Practices

- All code is reviewed before merging
- Branch protection rules enforce PR reviews
- Dependencies are regularly audited
- Automated security scanning via n8n workflows
