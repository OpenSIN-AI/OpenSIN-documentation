## What Is Sovereign Repo Governance?

Sovereign Repo Governance is OpenSIN's **contract-based** governance system: every repo in the fleet must carry explicit, machine-readable governance artifacts that define branch protection rules, PR-automation behavior, platform intake, and required-file manifests. The contracts live alongside the code — not in a separate wiki or doc site.

The governance system is built on three pillars:

| Pillar | File | Schema | Purpose |
|--------|------|--------|---------|
| **Branch & Merge Rules** | `governance/repo-governance.json` | `repo-governance.schema.json` | Branch protection, required reviewers, fail-closed semantics |
| **PR Automation** | `governance/pr-watcher.json` | `pr-watcher.schema.json` | PR-watcher hooks, credential scanning, escalation paths |
| **Platform Registry** | `platforms/registry.json` | `platform-registry.schema.json` | Which external platforms this repo integrates with |
| **Required Files** | `required-files.manifest.json` | — | Canonical 22-file manifest every SIN A2A repo must carry |

> **Source of truth:** Templates live in [`Delqhi/upgraded-opencode-stack`](https://github.com/Delqhi/upgraded-opencode-stack/tree/main/templates/governance) (`templates/governance/`). The [`Template-SIN-Agent`](https://github.com/OpenSIN-AI/Template-SIN-Agent) contains generated instances.

## Governance Compliance Levels

Every OpenSIN repo falls into one of three compliance tiers:

### Tier 1 — Canonical (Full Compliance)

Repos generated by the `create-sin-a2a-agent.mjs` factory in `OpenSIN-backend`. These automatically receive all governance artifacts from `Template-SIN-Agent/required-files.manifest.json`.

**Required artifacts:**
```
repository root
├── AGENTS.md                          # Agent operating manual
├── agent.json                         # Agent identity & capability contract
├── A2A-CARD.md                        # Public A2A card (Markdown)
├── required-files.manifest.json       # Canonical 22-file manifest
├── mcp-config.json                    # MCP server definition
├── package.json                       # Node.js dependencies (bun-compatible)
├── .well-known/
│   ├── agent-card.json                # A2A discovery endpoint
│   ├── agent.json                     # Agent metadata
│   └── oauth-client.json              # OAuth client configuration
├── clients/
│   ├── opencode-mcp.json              # OpenCode MCP client wiring
│   └── codex-config.toml              # Codex client config
├── governance/
│   ├── repo-governance.json           # Branch protection & merge rules
│   └── pr-watcher.json                # PR review automation config
├── platforms/
│   └── registry.json                  # Platform intake registry
├── n8n-workflows/
│   └── inbound-intake.json            # n8n workflow template
├── scripts/
│   └── watch-pr-feedback.sh           # PR watcher entrypoint
├── docs/
│   └── 03_ops/
│       └── inbound-intake.md          # Operational intake guide
└── cloudflared/
    └── config.example.yml             # Cloudflare tunnel template
```

**Status badge:** `✅ Canonical` — tracked via Project 18 (SIN Fleet Compliance)

### Tier 2 — Adopters (Migrated Existing Repos)

Existing repos that have adopted the governance contracts via backfill. These carry the same artifacts as Tier 1 but were not generated by the factory.

**Required action:** Run the backfill (see [Adoption Process](#adoption-process))

**Status badge:** `🔄 Adopting` — tracked via Project 18

### Tier 3 — Legacy (Non-Compliant)

Repos that predate the governance system. These are not blocked from operating but are flagged in Project 18 and must be migrated.

**Status badge:** `⚠️ Legacy` — tracked via Project 18

## Adoption Process

### Step 1 — Check Current Compliance

```bash
# Verify required-files.manifest.json exists
ls required-files.manifest.json 2>/dev/null && echo "MANIFEST OK" || echo "MANIFEST MISSING"

# Verify governance files
ls governance/repo-governance.json 2>/dev/null && echo "REPO GOVERNANCE OK" || echo "REPO GOVERNANCE MISSING"
ls governance/pr-watcher.json      2>/dev/null && echo "PR-WATCHER OK"      || echo "PR-WATCHER MISSING"

# Verify platforms registry
ls platforms/registry.json 2>/dev/null && echo "PLATFORM REGISTRY OK" || echo "PLATFORM REGISTRY MISSING"
```

### Step 2 — Copy Governance Contracts

```bash
REPO_SLUG="your-agent-slug"  # e.g., sin-code-plugin

# Clone the template source
TEMPLATE_REPO="$HOME/dev/upgraded-opencode-stack"

# Copy governance contracts
cp "$TEMPLATE_REPO/templates/governance/repo-governance.template.json" "governance/repo-governance.json"
cp "$TEMPLATE_REPO/templates/governance/pr-watcher.template.json"      "governance/pr-watcher.json"
cp "$TEMPLATE_REPO/templates/governance/platform-registry.template.json" "platforms/registry.json"

# Copy required-files manifest from Template-SIN-Agent
cp "$HOME/dev/Template-SIN-Agent/required-files.manifest.json" "required-files.manifest.json"

# Substitute repo slug placeholder
find governance/ platforms/ -name "*.json" -exec sed -i '' "s/{{REPO_SLUG}}/${REPO_SLUG}/g" {} \;
```

### Step 3 — Create Required Directories

```bash
mkdir -p .well-known clients cloudflared governance n8n-workflows/{,.gitkeep}
mkdir -p scripts docs/03_ops platforms
```

### Step 4 — Populate Platform Registry

Edit `platforms/registry.json` and list all external platforms this repo integrates with:

```json
{
  "platforms": [
    {
      "id": "huggingface",
      "type": "model_hosting",
      "owner": "huggingface.co",
      "access": "api_token",
      "intake": "manual-none",
      "compliance": "fail-closed"
    }
  ],
  "failClosedDefault": true
}
```

### Step 5 — Configure PR Watcher

Edit `governance/pr-watcher.json` to define review automation:

```json
{
  "checks": [
    {
      "name": "credential-scan",
      "tool": "gh-secret-scanner",
      "onFail": "block-merge"
    },
    {
      "name": "governance-contract",
      "tool": "file-existence",
      "requiredFiles": ["governance/repo-governance.json", "required-files.manifest.json"],
      "onFail": "block-merge"
    }
  ],
  "escalation": {
    "onFail": "notify-sin-zeus"
  }
}
```

### Step 6 — Verify Adoption

```bash
# Run schema validation
ajv validate -s "$HOME/dev/upgraded-opencode-stack/templates/repo-governance.schema.json" \
  -d governance/repo-governance.json && echo "REPO GOVERNANCE VALID"

ajv validate -s "$HOME/dev/upgraded-opencode-stack/templates/pr-watcher.schema.json" \
  -d governance/pr-watcher.json && echo "PR-WATCHER VALID"

ajv validate -s "$HOME/dev/upgraded-opencode-stack/templates/platform-registry.schema.json" \
  -d platforms/registry.json && echo "PLATFORM REGISTRY VALID"
```

## Project 18 — SIN Fleet Compliance

All governance adoption status is tracked in **[Project 18 (SIN Fleet Compliance)](https://github.com/orgs/OpenSIN-AI/projects/18)**. Each repo has a compliance item with one of four status values:

| Status | Meaning |
|--------|---------|
| `canonical` | Full governance scaffold present, generated by factory |
| `adopting` | Backfill in progress, partially compliant |
| `legacy` | Pre-governance repo, migration planned |
| `blocked` | Governance check failed, merge blocked |

Fail-closed transition rules are defined in [`project18-status-contract.md`](/governance/project18-status-contract).
