# Glossary

Complete glossary of OpenSIN terms and concepts.

## A

**A2A (Agent-to-Agent)** — Protocol for direct communication between agents without a central coordinator.

**Agent** — An autonomous AI entity that can process inputs, make decisions, and produce outputs using LLMs and tools.

**Agent Card** — A metadata document that describes an agent's capabilities, endpoints, and supported protocols.

**API Gateway** — The entry point for all external requests to the OpenSIN platform.

## B

**Bridge** — A connector that links OpenSIN to external messaging platforms (Telegram, Discord, etc.).

**Batch Processing** — Processing multiple requests or tasks simultaneously for efficiency.

## C

**Crew** — A group of agents working together on a task (term used by CrewAI, equivalent to Team in OpenSIN).

**Chain** — A sequence of processing steps where output of one step feeds into the next.

**Context Window** — The maximum amount of text (tokens) an LLM can process in a single request.

## D

**Deployment** — The process of making an agent or team available in a production environment.

## E

**Event** — A signal emitted by the system when something significant happens (agent created, task completed, etc.).

**Event Bus** — A pub/sub system for distributing events to subscribers.

## F

**Fleet** — A collection of all deployed agents managed by the OpenSIN platform.

## H

**HF Spaces** — Hugging Face Spaces, a platform for hosting ML applications (used for deploying OpenSIN agents).

## I

**Integration** — A connection between OpenSIN and an external service or platform.

## L

**LLM (Large Language Model)** — The AI model that powers agent intelligence (e.g., GPT-4, Claude).

## M

**MCP (Model Context Protocol)** — A standard protocol for connecting AI models to external tools and data sources.

**Memory** — The system that stores conversation history and context for agents.

**Middleware** — Software that sits between components to process requests and responses.

**Multi-Agent** — A system with multiple agents working together.

## N

**n8n** — A workflow automation tool used to orchestrate OpenSIN agents visually.

## O

**OMOC (Open Multi-Orchestration Controller)** — The swarm intelligence system for coordinating large agent fleets.

**Orchestrator** — The component that coordinates agent execution and team workflows.

## P

**Pipeline** — A sequential chain of processing steps.

**Plugin** — An extension that adds custom capabilities to agents.

**Prompt** — The input text given to an LLM to generate a response.

**Prompt Injection** — A security attack where malicious input manipulates an agent's behavior.

## R

**Rate Limiting** — Restricting the number of requests an agent can process in a given time.

**Registry** — A centralized catalog of all available agents and their capabilities.

## S

**Sandbox** — An isolated environment for running untrusted code safely.

**SDK (Software Development Kit)** — A library for integrating OpenSIN into applications.

**Streaming** — Sending response tokens as they are generated, rather than waiting for the complete response.

**System Prompt** — The initial instructions that define an agent's behavior and capabilities.

## T

**Team** — A group of agents working together with a defined strategy.

**Token** — A unit of text processed by LLMs (roughly 4 characters or 0.75 words).

**Tool** — A function that an agent can call to perform specific actions (search, calculate, etc.).

## V

**Vector Database** — A database optimized for storing and searching embeddings (used for agent memory).

## W

**Webhook** — An HTTP callback that notifies external systems when events occur.

**Workflow** — A defined sequence of steps to accomplish a task.
