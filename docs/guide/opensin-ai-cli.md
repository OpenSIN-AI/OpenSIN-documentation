# OpenSIN-AI CLI

The standalone `opensin-ai-cli` repository is no longer the primary recommended
entry point for new OpenSIN work.

> [!IMPORTANT]
> According to
> [`OpenSIN-overview/docs/CANONICAL-REPOS.md`](https://github.com/OpenSIN-AI/OpenSIN-overview/blob/main/docs/CANONICAL-REPOS.md),
> the canonical autonomous coding surface for new work is
> [`OpenSIN-Code`](https://github.com/OpenSIN-AI/OpenSIN-Code).

## Current Status

| Repo | Status | Recommended for new work? |
|------|--------|----------------------------|
| `OpenSIN-Code` | Canonical autonomous coding CLI | Yes |
| `opensin-ai-cli` | Rationalization pending | No |

## What This Means

- If you want to use the current OpenSIN coding stack, start with
  [OpenSIN-Code](/guide/opensin-code).
- If you are maintaining an existing installation that still depends on
  `opensin-ai-cli`, treat it as a compatibility surface, not as the product
  center.
- New features, new docs, and new integration guides should point to the
  canonical repos listed in `OpenSIN-overview`.

## Use Instead

- [OpenSIN-Code](/guide/opensin-code)
- [SDK Overview](/sdk/overview)
- [OpenSIN-AI Overview](/guide/opensin-ai-overview)
