# Commit Message Guidelines

> **Version:** 1.0 | **Date:** 2026-04-04 | **Owner:** Community Team | **Status:** ✅ Active

## Purpose

This document provides guidelines for commit message guidelines within the OpenSIN-AI community.

## Scope

These guidelines apply to all community members, contributors, and maintainers.

## Guidelines

### General Principles

1. **Be Respectful:** Treat all community members with respect and dignity
2. **Be Constructive:** Provide helpful, actionable feedback
3. **Be Inclusive:** Welcome contributors of all backgrounds and skill levels
4. **Be Transparent:** Communicate openly about decisions and changes

### Specific Guidelines

#### For Contributors
- Follow the established coding standards
- Write tests for all new features
- Update documentation for all changes
- Use conventional commit messages

#### For Reviewers
- Provide timely, constructive feedback
- Focus on the code, not the person
- Suggest improvements, not just criticisms
- Acknowledge good work

#### For Maintainers
- Respond to issues and PRs within 48 hours
- Label and triage issues promptly
- Keep the project roadmap updated
- Communicate release schedules

## Enforcement

Violations of these guidelines may result in:
- Warning from community moderators
- Temporary suspension of contribution privileges
- Permanent ban from the community

## Contact

For questions or concerns about these guidelines, contact the Community Team via Telegram.
